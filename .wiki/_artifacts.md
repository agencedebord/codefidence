# Wiki artifacts classification

> Which files are human-maintained, which are auto-generated, and which are transitional.
> Internal reference for contributors.

## Source of truth (human-maintained)

These files are written and maintained by humans. They drive all derived artifacts.

| File / directory | Purpose |
|---|---|
| `domains/<name>/_overview.md` | Domain documentation with frontmatter (confidence, memory_items, related_files, dependencies) |
| `domains/<name>/*.md` | Additional domain notes (same frontmatter schema) |
| `decisions/*.md` | Business decision records (date-prefixed filenames) |
| `_templates/` | Templates for new domains, decisions, and memory item schema |

## Derived artifacts (auto-generated, do NOT edit)

Regenerated by CLI commands. Manual edits will be overwritten.

| File | Generated by | Trigger |
|---|---|---|
| `_index.md` | `index.rs` | `rebuild`, `index` commands |
| `_index.json` | `index.rs` | `rebuild`, `index` commands |
| `_graph.md` | `graph.rs` | `rebuild` command |
| `.file-index.json` | `file_index.rs` | `rebuild`, `index` commands (built from `related_files` in notes) |
| `_candidates.md` | `candidates/render.rs` | `generate-candidates` command (init workflow) |

## Transitional (may be removed or restructured)

| File | Purpose | Status |
|---|---|---|
| `_needs-review.md` | Review queue for items needing human validation | Generated during `init` scan; not yet integrated into main workflow |

## Regeneration cheat sheet

```
codefidence rebuild          # regenerates _index.md, _index.json, _graph.md, .file-index.json
codefidence index            # regenerates _index.md, _index.json, .file-index.json only
codefidence generate-candidates  # regenerates _candidates.md from codebase scan
```
